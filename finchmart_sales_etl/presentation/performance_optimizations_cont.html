Performance Optimizations (continued)

4
Pre-Aggregations
Gold layer computes daily_sales, store_performance, top_products, customer_spending aggregations. Power BI queries read pre-aggregated data instead of scanning raw transactions, ensuring consistent calculations across all dashboard users. This approach eliminates redundant computation and reduces query complexity.
⚡ Benefit: Dashboard refresh time reduced from 30+ seconds to under 5 seconds
5
Coalesce for Exports
coalesce(1) combines partitioned output into single CSV file for Power BI import, simplifying data loading process for business users. Trade-off accepted: loses parallelism during write, but acceptable for small Gold layer datasets. Provides predictable file naming and eliminates need to merge multiple partition files.
⚡ Benefit: Single file output, predictable naming, easier Power BI integration
Combined optimizations enable scalability to 100K+ transactions/day with sub-second query response times and 80% reduction in compute costs
